# Decisions: AMPLIFY-409

## [2026-06-12] LiteLLM pass-through 403 — права задаются на уровне API-ключа

**Что произошло:** При вызове `/elevenlabs/v1/speech-to-speech/*` через LiteLLM возвращался 403, несмотря на то что маршрут был добавлен в `general_settings.pass_through_endpoints` в конфиге сервера.

**Почему:** LiteLLM дополнительно проверяет права доступа к маршруту на уровне **конкретного API-ключа**. Глобальная конфигурация pass-through открывает маршрут на уровне сервера, но каждый ключ имеет свой список `allowed_routes`. Если pass-through маршрут не добавлен в этот список — ключ получает 403.

**Решение:** В LiteLLM UI → Keys → выбрать нужный ключ → Edit → поле **Available Routes** → добавить:
```
/elevenlabs/v1/speech-to-speech/*
```
Wildcard обязателен: без него каждый конкретный voice_id нужно перечислять отдельно.

**Влияние на plan.md:** Нет — решение соответствует плану, просто добавлен нетривиальный шаг настройки.
